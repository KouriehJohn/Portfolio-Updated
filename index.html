IDE'S

Visual Studio, Sublime, etc.

Development Enviroments

I've tried my fair share of code editors from light weight, to Visual Studio. They all have their positives and negatives. Some get in your way, while others help but take a long time to set up.

I have tried a few, to name some -

  • Visual Studio Code
    - Very fast and easy to edit with. Like anything MicroSoft, it does have its moods.
  • Atom
    - A beautiful and helpful editor, altough its slower to load and sometimes its plugins are in my way.
  • Sublime Text
    - Probably my favorite editor, it's quick and doesn't get in your way. An added plus is its univeral across platforms.
  • Dreamweaver
    - A great tool with its own unique style and feel. Quite premium!

GitHub

Version Control

Version Control

Very familair and practiced with Github version control, what it does and why. I have done so using the command line for Windows 7 and 10 as well as Git's GUI. I prefer the command line. I have also practiced the using same process with a Linux operating system.

I have practiced a lot using Git in many ways -

  • Getting Started
    Installing Git, initializing a repository.
  • Cloning, Push, Pull
    Practiced cloning from repositories, pushing changes made from daily work flow as well as pulling from branches as needed.
  • Master and Branch control
    Finding yourself working on different features of the same project, its best to sperate the branches and merge to the master to avoid bugs.
  • IDE environment
    Though I prefer the command line, many of the IDE's I have worked with allow Git version integration which do make workflow smoother.
